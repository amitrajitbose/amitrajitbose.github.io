Introduction To Relational Models

Chapter Material

Relation - an unordered collection of attributes or fields or columns of a table. Attributes - each column is an attribute. Domain - set of possible values that an attribute can take. Attribute Value - the data in a certain row and column, corresponing to a certain attribute. They are normally atomic, i.e indivisible data type. Mathematically, relation schema is a subset of multiple domain sets, or we can say a relational schema is a collection of attributes.

Keys

The set of keys K, is a non-empty subset of relational schema, thus is a subset of attributes. Thus, a key can be one or more attributes. The values of each key uniquely identifies each row in the table, we can thus say it is a super key of R, i.e the relational schema. Super key is a candidate key if set K is minimal. There can be multiple candidate keys in a relation R, then we select one as the primary key. Surrogate key or synthetic key is the artificially generated key that is used as a key during the absence of any primary key. No key can be a nullable attribute. We underline an attribute in the schema to show that it is a primary key. Simple key is a key which consists of a single attribute. Just the opposite is called Composite Key, here multiple attributes together form a key. Each attribute in a composite key are not simple keys. When we make a tuple of multiple simple keys, then we can call it as a Compound Key. It is different from composite key. When we take a key of some other relations to make the key of another different relation, then we call such a key as Foreign Key.

Relational Query Language

It is a declarative programming paradigm.

Relational Operators

Select

This is an operation to choose a subset of rows based on certain conditions (propositional). In relational algebra it is denoted by sigma notation.

Project

This operation chooses a set of columns from the original table. Denoted by pi notation. Also, each row has to be distinct. Two same row vectors cannot be simultaneously present. Also, the project command cannot take less than one attribute, because zero attribute will not be able to build a schema.

Union

This is same as the set theory union. All records that exist in both the records will be put together in a single table. Denoted by U.

Set Difference

This is same as normal difference of two sets. Note that, R-S is not same as S-R. Denoted by minus symbol.

Set Intersection

This is same as usual intersection of two sets. Denoted by inverted U symbol. This is a derived relation, not fundamental. Because, R(intersect)S = R-(R-S)

Joining

Joining two relatons is same as taking cartesian product of the two sets. Denoted by x.

Renaming

Denoted by rho symbol(P). It is not a fundamental operation.

Composition

Combining multile operations and connecting them to yield some result.

Natural Join

Cartesian product of two tables such that the values for common columns/attributes of the two tables are identical. It is a derived operation. Basically, cartesian(joining) minus (those rows where the common attributes have different values).

Aggregate Operations

  • Count
  • Sum
  • Average
  • Maximum
  • Minimum
