How I Automated Git Commands On Linux

Everytime I boot in to my system, I realise there is so much freedom I got as a user and a developer after switching from Windows to Linux. I keep wondering, how brilliant Linus Tolvards was!

I love to maintain repositories for my ongoing projects so as to keep track of my work and also a backup (FREE), if you understand what I mean. If not, please lookup GitHub. So there was this project of mine, where I was the sole contributor (obviously, because it was private and only for my personal use) and everytime I made some modifications or additions, I had to write out these three separate commands to push it to the remote branch on the server.

git add .
git commit -m "Here goes the commit message"
git push

I realised that I am doing unnecessary work, the mind of an Engineer turned on to be precise. I figured out that we can convert this to a single command by joining each separate command with &&, thus executing a single command. Cool enough!

But still I had to look it up from Bash history or re-type always. So I wrote up this shell script and saved it as gitit.sh (Git It, that’s what I intended to mean), in my home directory.

git add . && git commit -m "$*" && git push

Then I added the following line at the end of the .bashrc file.

# custom alias
alias gitit='bash ~/gitit.sh'

You can edit bash runtime configuration with sudo nano ~/.bashrc

Voila! And that’s all! Now all I have to do is

gitit This is the commit message

in the project folder. That would do everything for me. I recommend using git status before commiting something and to use SSH to connect to GitHub instead of password based authentication.
